# Data Storytelling Templates & Techniques

> Extracted from SKILL.md — presentation templates, writing techniques, and visualization patterns.

## Visualization Techniques

### Chart Selection Guide

| Data Pattern | Best Chart | Example |
|-------------|-----------|---------|
| Trends over time | Line chart | Monthly revenue |
| Comparison | Bar chart | Sales by region |
| Part of whole | Donut/pie | Market share |
| Distribution | Histogram | Customer ages |
| Correlation | Scatter | Price vs. volume |
| Ranking | Horizontal bar | Top products |
| Flow/conversion | Funnel | Sales pipeline |
| Geographic | Map/choropleth | Regional data |

### Data-Ink Ratio

Remove non-essential elements: gridlines, borders, legends (when obvious), background colors. Every pixel should represent data.

### Annotation Patterns

- **Callout**: Arrow pointing to key data point with text
- **Reference line**: Horizontal/vertical line for benchmarks
- **Highlight zone**: Shaded region for context
- **Before/After**: Split visualization showing change

## Presentation Templates

### Executive Dashboard Template

```
THE HEADLINE
  ↳ One sentence: what happened and why it matters

KEY METRICS AT A GLANCE
  ↳ 3-5 KPIs with trend arrows (↑/↓) and vs. target

WHAT'S WORKING
  ↳ Top 2-3 wins with supporting data

WHAT NEEDS ATTENTION
  ↳ Top 2-3 concerns with impact quantified

ROOT CAUSE
  ↳ Brief analysis of the key driver

RECOMMENDATION
  ↳ Specific action with expected impact

NEXT MONTH'S FOCUS
  ↳ 2-3 priorities
```

### Story Frameworks

**Data Revelation (Investigation → Insight → Action):**
1. The Hook — surprising fact or question
2. The Context — what we know
3. The Problem — what the data reveals
4. The Insight — the "aha" moment
5. The Solution — recommended action
6. Expected Impact — quantified outcome
7. Call to Action — next steps

**Before/After (Transformation Story):**
1. Where We Started — baseline metrics
2. What Changed — intervention
3. The Transformation — results
4. Key Insight — why it worked
5. Going Forward — sustain/scale

**Comparative Analysis:**
1. The Question — what we're comparing
2. The Comparison — side-by-side data
3. The Analysis — drivers of difference
4. The Recommendation — which option wins
5. Risk Mitigation — addressing concerns

## Writing Techniques

### Numbers That Speak

| Instead of | Write |
|-----------|-------|
| "Revenue increased 23%" | "Revenue grew $2.3M — enough to fund the new team" |
| "NPS went from 32 to 67" | "Customer satisfaction doubled in one quarter" |
| "8,500 users" | "8,500 users — equivalent to a small city" |

### Narrative Connectors

| Transition | Examples |
|-----------|---------|
| Contrast | "However", "Despite this", "On the other hand" |
| Cause | "This drove", "As a result", "Consequently" |
| Emphasis | "Notably", "Most importantly", "The key finding" |
| Sequence | "First... Then... Finally" |

### Headline Formulas

- **Metric + Direction + Context**: "Q3 Revenue Up 23% — First Growth in 4 Quarters"
- **Action + Impact**: "Reducing Churn by 5% Would Save $2.1M Annually"
- **Comparison**: "Mobile Users Convert 3x More Than Desktop"

## Best Practices

1. **Lead with the insight**, not the methodology
2. **One message per slide** — if you need more, add slides
3. **Use consistent color coding** — green=good, red=bad, blue=neutral
4. **Always show context** — vs. last period, vs. target, vs. benchmark
5. **Quantify everything** — "significant" is meaningless without numbers
6. **End with a decision** — every story should drive action
